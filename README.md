# Supermercado_Concurrente
Simulación de cobro en un supermercado desarrollada en Java utilizando programación concurrente e hilos. El sistema permite procesar múltiples clientes y cajeras simultáneamente, calculando tiempos de atención, productos comprados y costos totales de cada compra.

# EA2: Optimización del Proceso de Cobro en un Supermercado mediante Hilos y Concurrencia
 

# Objetivo:
Aplicar los conceptos de concurrencia en programación con Java para mejorar la eficiencia de un sistema que involucre procesos simultáneos

 

# Instrucciones:
Para el desarrollo de la actividad se debe realizar lo siguiente:

Leer el documento Caso_de_Estudio.pdf.

Diseñar y construir una aplicación en Java que simule el proceso de cobro en un supermercado.

Aspectos a tener en cuenta para construir la aplicación:

El software debe permitir la creación de múltiples cajeras y clientessimultáneos

Debe simular el proceso de cobro, calculando el tiempo necesario para cada compra

Debe mostrar el tiempo total de cobro para todas las compras

Mostrar los productos comprados y el costo total de cada producto, así como el tiempo en procesar la compra de dicho producto
